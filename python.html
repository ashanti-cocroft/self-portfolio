Variables & Data Types

‚úì MASTERED

WHAT_I_LEARNED

Variables are containers for storing data values. Python is dynamically typed, meaning you don't need to declare variable types explicitly. I've mastered how to work with all fundamental data types and understand when to use each one.

  • Variables store references to objects in memory
  • Python uses duck typing - if it looks like a duck and quacks like a duck, it's a duck
  • Mutable vs immutable types affect how data is passed to functions
  • Type conversion and casting for flexibility

Example 2: Collections 

# Lists - ordered, mutable
python_skills = ['variables', 'loops', 'functions', 'OOP']
python_skills.append('Django')  # add items
python_skills[0] = 'advanced variables'  # modify

# Tuples - ordered, immutable
coordinates = (31.3271, -89.2903)  # Hattiesburg!

# Dictionaries - key-value pairs
portfolio = {
    'name': 'Ashanti',
    'skills': ['Python', 'Django', 'JavaScript'],
    'projects': 12,
    'github': 'github.com/ashanti'
}

# Sets - unordered, unique values
completed_courses = {'Python', 'Git', 'SQL'}
completed_courses.add('Django')

HOW_I_CAN_EXPLAIN_IT

Think of variables as labeled boxes. Each box (variable) has a name on it and can hold different types of items (data). A string is like a note with text, an integer is a whole number, a float has decimals, and a boolean is a yes/no answer.
